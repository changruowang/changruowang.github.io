实验记录

发表于 2021-05-11 更新于 2021-08-27 分类于 图像预处理 ， 虹软实习笔记

图像提亮

第一个数据集实验记录

  1. restore_gan : 和EnlightenGAN一样。 输入是 31x31均值滤波的ref图和原图 保留了原文的注意力

    • 首先对于原本比较亮的区域没有过曝到较好的效果，可能是这个注意力导致的
    • GAN的效果好像并不好

  2. restore_spa_exp ：去掉了GAN，输入为 ref模糊+原图，损失：亮度损失+结构一致性损失+VGG损失。

    • 没有很明显的鬼影 边缘处还是有点泛白 特别对于运动区域和背景区域颜色差异大的地方
    • 相对直方图匹配的优点是，在部分区域噪声不那么明显

  3. restore_gan_spa_exp : 对抗损失+0.5亮度损失+结构一致损失 (去掉了unet里的注意力)

    • 相对没有GAN的方法2 对运动区域的皮肤增强额效果更自然 ，测试图中有个很暗的图也没明显的色偏差
    • 但是对亮的区域增强不是很彻底，例如测试图中天空区域，参考图的云层是完全看不到的 但是增强结果还能看到云的轮廓说明提亮不彻底

  4. restore_line_trans: 相对2 restore_spa_exp 将输出调整为输出线性变换的形式 即网络预测 一个 3x4通道的变换参数，并在损失中对参数添加 tv 损失

    • 和方法2效果差别不是很明显。但是从训练损失上看没有方法2收敛的低

  5. restore_cur_trans: 和4的区别就是输出预测 3x8=24通道的变换参数，执行 ZeroDEC 文章中的曲线变换，主干网络也是使用ZeroDEC中的（使用Unet会梯度爆炸）。增加tv损失。

    • 效果很奇怪，和文中消融实验中没有 tv 损失的效果一样。因此考虑增加tv损失的权重

  6. restore_mask_loss: 和2的区别是

    • 输入为 低光照图 和 ref原始图（不模糊）
    • 计算损失的时候加上了mask mask为1的区域是通过预先计算的可能的运动区域。 对于非运动区域，给参考图较大的损失权重，对于疑似运动区域，给与较大的一致性权重抑制鬼影
    • 同时给与运动区域和直方图匹配的结果较大的MSE损失（如果不加这个训练出来运动的手部区域比较黑，所以这块就用直方图匹配的结果近似了）

新增qc数据集后

使用新增数据集qc后 首先是里面有大量的对齐的图片，训练之后有重影，训练损失如下图。考虑会不会是数据集中对齐图像太多的原因。（右边是在最开始数据集上的训练结果，可以看出损失波动较小）

于是手动删掉了所有对齐图像，只剩500对左右的运动图像结果使用这个数据集训练之后在10 epoch左右就nan了。并且起始的MSE损失很大，而且运动区域的损失远大于非运动区域的损失

  1. 尝试降低学习率无效
  2. 关闭SPA损失无效
  3. 关闭运动区域损失也无效。
  4. 降低MSE损失也无效。
  5. 可能是 计算MSE的时候基数使用的 mask的数量 导致 nan 的？换成常规的损失函数还是会nan

如果某一个损失函数写的有问题导致计算的时候出现 nan 那个即便是该项损失权重 * 0 ，它还是会导致损失崩溃。因此调试的时候不能想要通过0权重来屏蔽可能出现nan的损失

修改之后 出现的问题

  1. 会有阴影，训练中会有奇怪的 条纹？这是从哪儿来的？学习率设置0.001
  2. 并不是所有的运动图象都有阴影，有些图象运动也很大 但是没有阴影。难样本挖掘试试
  3. 发现 一个batch里总会有几个图测试的 一致性损失很大 10 左右 其他的很小 1 左右。输出发现大部分很梯度损失很大的都是图中运动区域很少的图产生的。所以原本训练16个batch平均下来 spa损失只有1左右，但是用难样本提取出来的一致性损失就都是10左右，且提取出来的大部分都是对齐的图象。
  4. 所以由于mask有噪声，导致计算的 梯度 损失 有误差，正确的梯度方向被 错误的梯度方向淹没了？
  5. 用mse损失筛选出来的也 大都是对齐的图象 ???
  6. 去掉 梯度损失 只用 mse损失 看看 ????
  7. 测试一下出问题的　图象　哪项损失　比较大　　得　保证损失可以在图象中体现出来　直接可视化　每个像素的损失
    • MSE损失可以体现出来 但是 训练的时候处于震荡的状态
    • 将损失归一化后 梯度损失 不能体现出 有伪影的地方。梯度难以区分噪声和阴影
    • 仅使用 mse1（对齐区域） mse2（运动区域）损失训练后 虽然ms2损失 有所降低。但是运动区域的视觉效果变化不大。随着ms2损失的下降，运动区域的梯度损失有一定降低（降的不多）
  8. 测试 输入 为 低光照 和 参考图的时候 网络是不是可以更好的检测运动区域

可能的解决方法

目前的途径是让网络 隐式 的学习运动运动区域，然后把参考图照搬过去。但是原本的端到端的思路在新的数据集上效果不好。

  1. 可能是网络无法区分噪声和运动像素，导致损失无法收敛。

    • 增加可变形卷积 增加网络对运动区域的判别能力
    • 增加 特征 注意力机制 注意力图可以通过mask适当的约束 （可以约束静止区域呀 如果 注意力图给于了静止区域较小的权重，就惩罚，但是这样会不会生成全图都很大的注意力？ ）

  2. 现有的损失可能无法 很好的指导网络区分少部分的运动像素（毕竟还是有其他运动区域没有问题）。那么可能是 mask的精度不高，或者损失函数设计不好

    • 手工标注精度更高的 mask
    • 通过其他监督的方式约束 mask 例如运动区域都有很强的 语义信息 大部分都是人 可否通过这个途径约束 运动区域的判别？

  3. 模糊参考图输入 使用端到端的思路

  4. GAN 我认为 单单使用GAN效果一定是不如上述方法好的。将GAN作为辅助手段较好。

    • 条件GAN , 将参考图旋转平移生成条件图象

调整qc_mask实验记录

尝试将 mask 手工调整后 可以训练出没有明显阴影的结果。

调参

尝试将 背景梯度损失权重加大 然后通过其他 途径抑制运动阴影，来使得结果更清晰。

  1. 加入 余弦相似度损失 作用在运动部分。 可视化表明余弦损失 对 阴影要比 MSE敏感，主要体现在同一幅图余弦损失在背景噪声处的相似度小，在运动阴影处的相似度大。
  2. 初始学习率 设置 0.001虽然其实损失振荡比较大 设置成 0.0001学习率起初下降挺快 但是就会有奇怪的波纹 不知道训练久了会不会消失
  3. 尝试了 不加背景梯度损失 背景梯度损失权重0.2 背景梯度损失0.4+运动区域cos损失（压制运动阴影） 从前到后 图象运动阴影逐渐加重，中间那个阴影效果尚可接受。但是背景噪声和恢复效果 从前到后也一次提高。

难样本挖掘

尝试通过 难样本挖掘 提高清晰度 （要抑制运动阴影） 因此在上面 0.4 背景梯度的权重下预训练的模型基础上，将运动区域余弦损失设置为难样本判别依据，每个epoch选出前50%的难样本更新梯度。最后阴影效果有明显好转。

生成网络结构调整

生成网络尝试其他结构

可变形卷积

  1. 在UNET 前面加一层可变形卷积，训练时候的收敛速度还没有UNET快，并且最终阴影比单纯使用UNET要严重。这表明加入可变形卷积反而增加了训练难度。 （但是这种加法不太好，其他网络前置都是堆4，5个卷积层分别提取两张图象的特征 然后做特征对齐，本次尝试前置就一个特征提取，因为后面还有UNET）
  2. 将UNET 和 可变形卷积结合。使用共享权重的编码器分别对两张图做特征提取，并且在编码器的每个尺度上完成对参考特征做特征对齐，且可以自底向上级联。最后在UNET跨层连接时，将跨层的特征替换为特征对齐后的特征。这种方案占用显存较多，随便训练了 几 个epoch 好像不太好收敛
  3. 完全使用EVDR这汇总方案，不使用UNET这种结构。不过中显存不够。需要简化结构，未尝试。

自定义attention网络

首先分别对低光照图和参考图分别作 特征提取 然后 生成 attention_map 对两个特征做融合。最后解码输出。（输出不能再设置为残差的形式）

  • 为了提高 mask的运动检测精度，输入为初步提亮后的图象 和 参考图像，这样输出就不能用残差的形式（否则会完全拟合到匹配enhance的结果上） 同时 假定大部分图上 运动区域占少数，给mask加上惩罚。
    • 可以明显突出 运动像素 但是 不是很彻底 会有冗余 相同的损失占比 比之前 对运动阴影抑制效果好
    • 最终的结果有点太依赖于 mask这个过程估计的结果的好坏，如下图。明显可见匹配增强的结果被带入了最终结果。
    • 那么mask 的精度问题又是 一个 悖论 如果才能分割出高精度的mask呢？

  • 不共享权重，输入为低光照图 和 参考图 防止把 匹配的结果照搬 让低光照路径的特征提取网络学习提亮属性。但是这样对运动检测会困难，因为两个图不在一个亮度上 需要隐式提亮后attention。
    • 直接一种损失函数端到端的学习，可以学习到很好的mask 但是对maks的运动区域亮度没提起来。并且 mask 边缘过渡的不是很好

GAN网络

CGAN / RaGAN / mix augue 的尝试

  1. 判别器输入 ref + 非对齐的正常图象，损失WGAN，判别器结构patchGAN，输入参考图模糊。效果还可以 但是为啥 判别器损失不收敛。难道是生成器太弱了？生成的图象噪声很明显，细节也模糊了。
  2. 同样的配置 将2的输入换位 ref + ref(旋转小角度) 后 ，WGAN的判别损失很快就降低到0，但是生成图象的质量也并未提高。将判别器换为结构更为复杂的 UNET 也没用。
  3. 常规损失 + WGAN损失 没啥特别的效果

补全网络

对上面 attention 网络的一个衍生。

之前使用的 mask损失 在 -3ev这种噪声很严重的数据集上效果又不行了，手工标注 运动区域也不行。因为mask loss 的本质是由 运动区域前景和背景的差异生成的。

  • 因为图象噪声很大，拉亮后背景区域和参考图的背景区域像素值差异也比较大，另一方面，产生鬼影的区域，前景背景的差异也许反而并不那么大了，就导致网络无法区分运动像素和背景噪声（猜想），所以导致-3ev数据集上mask loss方法更难训练。增加运动区域权重是可以但是 未做进一步尝试
  • mask损失并不能使网络学习到语义信息，因为他的来源是前背景像素差异，而这个差异仅仅来源于不对齐的边缘部分，不能例如手有少量位移，那么产生损失的地方只有位移那个区域。（也不一定）

attention网络的优点

  • 一方面可以通过对预测的mask添加强监督损失，使得mask能都分割出语义信息，可以克服噪声混淆运动像素的问题
  • 如果mask预测的运动区域是完备的，那么通过特征融合 抹掉参考图中不需要的信息，就可以保证最后结果没有鬼影的问题

缺点

问题

  1. 上面一排 明显 直接 两个特征直接乘mask融合 提亮的区域结果 和 原始图象结果 有色差，这个很难解决
  2. mask预测的面积太小了 盖不住，因此需要确保mask的完备性

尝试的方法

噪声估计

  • 问题1：样本类别不均衡 Cnoise类别中尤为明显，Color样本占了百分之77的比例

    1. 对Cnoise类样本过采样和降采样
    2. 集成方法，从大量样本中随机选取一部分数据和少量标签的样本组成一个训练集。这样抽样成N个数据集，每个数据集分别训练一个分类器共同判定
    3. 正负样本设置不同的 损失权重

  • 问题2：评价指标

    1. 验证集也存在类别不平衡问题的 直接用精确率做指标不行
    2. 三个属性不能割裂来看，要比较三个属性同时正确时的准确率、至少两个属性预测正确、仅有一个属性预测正确时的正确率（并且得在单个属性中标签类别平衡的情况下计算，否则准确率指标没意义）
    3. 综上，将一张图的三个预测属性结果和标准比较，记预测正确的属性个数x x>阈值th 为正，否则为负。然后计算

  • 问题3：三个属性的独立性和相关性。

    1. 如果一张图的三个属性是相互独立的，那么使用同一个网络输出三个属性的分类结果，每个属性分别计算交叉熵损失
    2. 如果三个属性间不相互独立，需不需要通过一个损失约束呢？就像评价指标中，需不需要保证单张图像预测的三个属性尽可能多的正确。

  • 问题4：3000x3000的图一个iter里只使用了一次，见的太少了

Crop感兴趣区域的方法

  • 梯度图求 局部熵
  • 首先边缘检测，然后计算3x3局部窗口的标准差，并剔除含有边缘的窗口。最后计算64x64 patch 中标准差的中位数，如果中位数大于阈值，则说明该有噪声或着纹理丰富。
    1.  边缘检测尽可能地去掉边缘 但是可能会同时去掉强度较大且颗粒较大的噪声，因为他们轮廓大，强度大和边缘难以区分。如果不去掉边缘，那么方差块中存在的边缘也会导致计算的局部方差较大。也可能去掉像树叶这种密集的区域
 2.   剩下的没被边缘检测检出的 应该就是纹理区或者噪声区域

UNET-GAN

A U-Net Based Discriminator for Generative Adversarial Networks

主要思想

文章的工作主要是针对 GAN 的判别器的改进。使用一个U-NET作为判别器，判别器的编码部分输出对图象的全局分类结果，判别器的解码部分输出对图象像素级的判别。因此这个判别器既能判别全局也能判别局部信息。同时作者结合了 最新的正则化技术 MixCup 提出了个一致性正则化损失 。

主要内容

研究背景

GAN 目前从三个研究角度进行，即大规模训练、网络结构设计和正则化技术。尽管近年来取得了很大的进展，但如何综合具有全局语义连贯、长程结构和细节准确性的图像仍然具有挑战性。对抗器的好坏会直接影响生成器的效果。在目前最先进的GAN模型中，作为分类网络的鉴别器只学习一种表示，因此它通常要么关注全局结构，要么关注局部细节。合成样本的分布随着训练过程中生成器的不断变化而变化，容易忘记之前的任务 (在判别器训练的背景下，学习语义、结构和纹理可以被认为是不同的任务) …..

结构

结构很简单 ，使用UNET 判别器可以同时输出去全局和局部判别。对于全局判别的标签就是和输入图像一样的 0/1的值。除此之外，还是用了MixCut正则技术

CutMix

mixcut用在对判别器的训练。通常判别器输入要么是正样本要么是负样本。而MixCut就是复制粘贴的方式结合在一起输入网络（M就是二值掩膜）。对应的标签按照 正负样本 面积比例给的。除此之外还有 Mixup,Cutout

  • Mixup:将随机的两张样本按比例混合，分类的结果按比例分配；
  • Cutout:随机的将样本中的部分区域cut掉，并且填充0像素值，分类的结果不变；
  • CutMix:就是将一部分区域cut掉但不填充0像素而是随机填充训练集中的其他数据的区域像素值，分类结果按一定的比例分配

CutMix优点：

  1. 在训练过程中不会出现非信息像素，从而能够提高训练效率；
  2. 保留了regional dropout的优势，能够关注目标的non-discriminative parts；
  3. 通过要求模型从局部视图识别对象，对cut区域中添加其他样本的信息，能够进一步增强模型的定位能力；
  4. 不会有图像混合后不自然的情形，能够提升模型分类的表现；
  5. 训练和推理代价保持不变

LIME 传统光照分解

优点 ：

  1. 图像分解效果较好 图像增强的最终结果不会太糟，对一般的图像都有普适性

缺点：

  1. 使用DMB3去噪效果远不如深度学习方法的效果
  2. 会有一些颜色失真的情况 也没办法应对
  3. 光照调节不够灵活 只能通过r矫正来 改变光照程度 并且也会有一定的过曝

KinD Net 基于深度学习的方法

优点：

  1. 图像分解网络不需要大量的高质量图片训练 只需曝光度不同的图片就可以完成整个网络的训练
  2. 光照调节灵活
  3. 含有针对特定数据集的去噪重建网络 这一点效果相对LIME直接使用DMB3效果要好很多

缺点：

  1. 在其他未fine turn数据上的光照分解网络性能 <= LIME 根据约束优化分解网络 在不使用大量优质参考图像训练的条件下 不如直接采用传统方法估计光照 这样估计的结果肯定不会太差 例如仅在LOLDataset上训练的分解网络在其他数据上效果较差
  2. 图像分解由于没有直接满足 I = L*R LIME中若 I = 0 I/L= R必定等于0 而 kind中 I != LR 只是近似 因此 当 I = 0 R 依然会趋于过曝 当然二者都有不同程度的过曝 因此LIME对黑色的区域分解结果比LIME还差

无监督图像增强方法

一些想法

  1. 因为传统图像增强方法例如直方图均衡化 他们可以作为先验知道CNN约束
  2. CNN中 应用 Retinex 模型的优点在于基于这种先验 可以使用更小的网络 来完成任务 而直接端到端的增强方式 需要更大的网络去拟合，也就是先验的存在降低了对网络拟合能力的需求
  3. 但是Retinex针对过曝的图像不友好，因为I./R的原因。而有的文章使用 曲线的方式 来增强 网络输出曲线的参数，不同像素位置的曲线不同，那么他的调整效果不同，具体的 可以增强也可以压缩像素值，而这一点是Retinex做不到的。但是感觉这种学习曲线参数和使用CNN直接端到端的映射区别不大呀。可是依然使用小网络实现了不错的效果。
  4. 借鉴图像先验来设计损失函数 例如 Gray-World color constancy

可以考虑的出发点

  • 可不可以设计 “多专家” 的网络学习过程，即多种传统方法的增强结果做损失函数约束 + 基于图像理论设计的损失函数 来共同优化网络的学习，增加的第二项目的是 根据传统增强方法中存在的 缺陷而设计的损失函数。这样出来的结果会不会优于传统方法。

    1. 第一项的损失是为了约束网络的学习方向，可以考虑计算训练过程中的 结果来 动态 调整各项损失权重
    2. 通过什么方法 改善 Retinex 结构 的缺点？

  • GAN类方法的优点

  • 使用对偶估计的方法，来加强对过曝光的适应性，将传统方法使用CNN实现

    1. 文中 对反转的输入图像做光照估计 可以很好的抑制过曝光区域，将 三张图融合 来达到更好的效果
    2. 但是这么做 算法的实时性不行，一方面本来传统的光照估计方法速度不是很快，如果做两遍更浪费时间，另一方面，它融合需要计算各个图像的质量然后计算融合权重，这也是很耗费时间的步骤。
    3. 使用CNN来改造有如下好处：
      1. 尽量用一个网络来同时对输入图像和反转图像进行光照估计，减少计算量。
      2. 使用CNN来学习三张图的融合权重，自适应的目的 会不会效果更好。
      3. 可以根据已有数据来离线学习，效果会更好？
    4. 光照估计可以考虑全局 和 局部特征提取，KinD 的照明估计 由于学习的是局部小patch信息 而传统CNN的方法是全局优化 看起来更自然。
